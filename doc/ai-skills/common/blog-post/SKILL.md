# Blog Post Skill - 博客文章处理规范

## 适用范围

本 skill 适用于处理本项目中与博客文章相关的所有任务，包括：
- 创建新博客文章
- 编辑/修改现有博客文章
- 格式化文章内容
- 优化文章 SEO
- 管理文章标签和分类

## 执行流程

### 1. 任务分析与确认

- 首先明确用户需求：是创建新文章还是修改现有文章
- 若创建新文章，确认以下信息：
  - 文章标题（必填）
  - 发布日期（默认当前日期）
  - 文章标签（可选，建议至少1个）
  - 文章分类（可选）
  - 内容格式（Markdown）

### 2. 文件位置与命名规范

- 文章存储目录：`_posts/`，按年份分类
- 文件名格式：`YYYY-MM-DD-文章标题.md`
- 示例：`2023-10-15-如何高效阅读一本书.md`

### 3. 文章内容规范

#### 3.1 前置元数据（Front Matter）

所有文章必须包含标准的 YAML 前置元数据：

```yaml
---
layout:     post
title:      "文章标题"
subtitle:   "副标题（可选）"
date:       2023-10-15 12:00:00
author:     "作者名（默认：enilu）"
header-img: "img/post-bg-2015.jpg"  # 可选，默认使用网站配置
catalog:    true
tags:
    - 阅读
    - 效率
---
```

#### 3.2 正文格式

- 使用 Markdown 格式编写
- 支持 GitHub Flavored Markdown (GFM)
- 代码块使用三个反引号 ``` 并指定语言
- 图片存储在 `img/` 目录，使用相对路径
- 引用使用 > 符号
- 标题层级使用 # 符号（最多三级）

#### 3.3 读书笔记类文章规范

对于读书笔记类文章，除了遵循以上格式规范外，还应满足以下要求：

1. **深入说明**：在总结书籍内容时，应至少深入说明3到5个方面的内容，避免仅对书籍结构进行大致描述。可以选择书籍的核心概念、主要观点、方法特点、应用场景等方面进行深入分析。

2. **行文风格**：建议采用幽默风趣的风格，使用比喻、实例等生动的表达方式，避免枯燥无味的陈述。让读者在轻松愉快的氛围中理解和吸收知识。

3. **结构建议**：
   - 书籍简介：介绍书籍的基本信息和作者背景
   - 书籍特点：总结书籍的特色和优势
   - 核心内容深入分析：选择3到5个方面进行详细说明
   - 读书感悟：分享阅读过程中的心得体会
   - 学习建议：给读者提供学习方法和建议
   - 总结：对书籍进行总体评价和推荐

4. **实例参考**：可参考 `_posts/2026/2026-03-15-《动手学深度学习》读书笔记.md` 这篇文章，学习其写作风格和结构。

#### 3.3 示例结构

```markdown
---
layout:     post
title:      "如何高效阅读一本书"
subtitle:   "分享我的阅读方法"
date:       2023-10-15 12:00:00
author:     "enilu"
catalog:    true
tags:
    - 阅读
    - 方法
---

## 引言

这是一篇关于如何高效阅读的文章...

## 阅读方法

### 1. 预览法

在开始阅读前，先预览...

### 2. 做笔记

阅读时做笔记很重要...

## 总结

通过以上方法，你可以提高阅读效率...
```

### 4. 实施与验证

- 创建或修改文章时，确保格式正确
- 检查前置元数据是否完整
- 验证 Markdown 语法是否正确
- 预览文章效果（本地运行 jekyll serve）
- 确保图片路径正确

### 5. 提交规范

- 提交信息应清晰明确
- 示例：`Add: 如何高效阅读一本书` 或 `Update: 优化文章排版`

## 相关文件与资源

- 文章模板：可参考 `_posts/` 目录下的现有文章
- 图片资源：`img/` 目录
- 配置文件：`_config.yml`（包含站点配置）

## 常见问题

### Q: 如何添加图片？

A: 将图片保存到 `img/` 目录，然后在文章中使用相对路径：

```markdown
![图片描述](/img/post-bg-book.jpg)
```

### Q: 如何设置文章封面图？

A: 在前置元数据中设置 `header-img` 字段：

```yaml
header-img: "img/post-bg-2015.jpg"
```

### Q: 如何添加标签？

A: 在前置元数据的 `tags` 数组中添加：

```yaml
tags:
    - 阅读
    - 方法
    - 效率
```
